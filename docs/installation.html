Clean Installation

To perform a clean installation click on the Clean installation option.

Before you run the CMScout installer, please ensure that you have created a database, and a user that has permissions to access that database.

You will then be presented with a screen that shows what requirements CMScout 2 needs to install, ensure all the requirements are satisfied before continuing with the installation.

Follow the simple guided instructions and fill in all the required details.

If this is the first time that you are installing CMScout 2, it is recommended that you install the sample content as it will serve as a starting point for your own website.

Once you have entered all the required information, and agreed to the license agreement, click on install. The CMScout installer will the proceed to install the database tables, and setup the configuration file. Once this is done, you should delete the install directory for security reasons, CMScout will refuse to start if the directory is still there.

Mirgrate from 1.x

Before you begin the migration process, please ensure that you have a up-to-date backup of all the files and the database for your site. Your site is required to be version 1.23 or 1.24 for the migration script to work.

In order to have the migration work the best please delete all files and folders in the directory that your CMScout 1.x site was installed, except the following folders:

  • photos/
  • avatars/
  • downloads/
  • images/

These folders contain files that you have uploaded to your website.

After you have deleted all the files and folders, upload the extracted CMScout 2.x package that you downloaded to your website. If you are asked to overwrite any files, say yes.

Point your browser to your websites address.

To migrate from a existing CMScout 1.x installation, click on the Migrate from CMScout 1.x button.

Ensure that the permissions are correct, and click on next.

Enter in your existing database information and click on next.

The migrator will then check to see if there is a CMScout 1.x installation in the directory. You will then be required to select who the main administrator of the website is, and which group is the administration group, and which group is the default user group. Once you have selected the groups and users click on next, and then click on install.

After it has installed, please delete the install directory before continuing to use your website.

Note: Due to large changes between CMScout 1.x and 2.x you will be required to add or update the following items:

  • Forum Access Rights
  • Authorizations
  • RSS Feeds
  • Sub Site and Group Site menu items
  • Frontpage items
  • Menu items will need to be published

Upgrade

In order to upgrade a existing CMScout 2.x installation to the latest version (For example, version 2.00 to version 2.01) download the changed files package for your version of CMScout.

Extract the package, and upload the files to your website.

Goto http://www.yoursite.com/install and click on Upgrade existing CMScout 2.x1 installation to CMScout 2.x2 (Where 2.x1 is your current version, and 2.x2 is the version you are upgrading too).

The upgrade script will check to make sure that you can upgrade your installation.

If you can, click on the Upgrade button.

Your installation should then be upgraded. Delete the install directory to continue using your newly upgrade CMScout website.
